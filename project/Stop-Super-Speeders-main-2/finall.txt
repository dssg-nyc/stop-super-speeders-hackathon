## WHY IT'S HARD WITHOUT THIS SYSTEM
================================================================================

### The Math Problem: Trailing Windows

**24-month trailing window for drivers (11+ points):**
- Every day, the "24 months ago" cutoff moves forward by one day
- Old violations drop off; new ones come in
- A driver at 10 points yesterday might be at 11 today, or might drop to 9
- Tracking this manually for 500,000+ drivers is literally impossible

**12-month trailing window for plates (16+ tickets):**
- Same problem for vehicles
- Plates can be shared, sold, reassigned
- Different drivers can rack up tickets on the same plate
- You need to aggregate by plate AND track the window

### The Data Fragmentation Problem

**NY has 1,800+ local courts:**
- Each court uses different systems, formats, schedules
- Some use paper, some use ancient software, some use modern systems
- No single database for "all NY traffic violations"
- NYC has its own Traffic Violations Bureau (TVB) with different data format
- State Police tickets go through yet another system

**Without our system:**
- DMV would need to manually collect data from 1,800 sources
- Then merge, deduplicate, and aggregate
- Then calculate trailing windows
- Then identify threshold crossings
- Then send notices
- Then track compliance
- This would take an army of clerks â€” and still have errors

### Why November Stats and "New Triggers" Matter

**Monthly reporting requirements:**
- DMV must report how many drivers/plates triggered ISA each month
- "How many people crossed the threshold in November specifically?"
- This is different from "how many are currently over threshold"
- Requires tracking WHEN each driver first crossed, not just IF they're over

**Operational importance:**
- "New triggers" = people who just crossed and haven't been notified yet
- You don't want to re-send notices to people who got them last month
- Monthly batching aligns with DMV's existing workflows

## HOW OUR SYSTEM ACTUALLY WORKS (WORKFLOW)
================================================================================

### Step 1: Data In â†’ Statewide Understanding

**NYC Open Data (100,000 violations):**
- Source: data.cityofnewyork.us (Moving Violation Summons)
- Script: `ingest.py`
- Fields: plate, violation_code, date, coordinates
- All NYC DOF (Traffic Violations Bureau) records

**NY State Open Data (500,000+ violations):**
- Source: data.ny.gov (Traffic Tickets Issued: Four Year Window)
- Script: `generate_ny_state_violations.py`
- Fields: police_agency, court, county, violation_code, date
- Coverage: ALL 62 NY counties, ALL local courts, ALL police agencies

**Local Court CSV Upload:**
- Endpoint: POST /api/dmv/courts/upload
- Accepts standardized CSV with violation data
- Immediate processing and deduplication
- Courts can see their data reflected in real-time

**Database Storage:**
- All violations stored in unified `violations` table
- Schema tracks: driver_license_number, plate_id, violation_code, date_of_violation,
  ticket_issuer, police_agency, disposition, coordinates
- Deduplication via unique indexes on (plate_id, plate_state, date_of_violation)

### Step 2: Policy Logic â†’ Identifying Who Needs ISA

**Points per Violation Code (from isa_policy.py):**
- 1180A: 2 points (1-10 mph over)
- 1180B: 3 points (11-20 mph over)
- 1180C: 5 points (21-30 mph over)
- 1180D: 8 points (31+ mph over â€” most severe)
- 1180E: 6 points (school zone)
- 1180F: 6 points (work zone)

**ISA Thresholds:**
- Drivers: 11+ points in 24-month trailing window â†’ ISA REQUIRED
- Plates: 16+ tickets in 12-month trailing window â†’ ISA REQUIRED
- Either condition triggers the requirement

**Warning Bands (Proactive Outreach):**
- Drivers: 8-10 points in 24 months â†’ "Approaching threshold"
- Plates: 12-15 tickets in 12 months â†’ "Approaching threshold"
- DMV can warn these people BEFORE they hit mandatory ISA

**Status Computation (compute_status function):**
- ISA_REQUIRED: points >= 11 OR tickets >= 16
- MONITORING: points >= 6 but < 11
- OK: below monitoring threshold

### Step 3: ISA Lists â†’ Operational Use

**ISA List Tab (Frontend):**
- Shows current counts: "X drivers â‰¥11 pts" and "Y plates â‰¥16 tickets"
- November stats badges: "X drivers triggered in Nov", "Y plates triggered in Nov"
- Two main tables:
  - Drivers table: license, name, violations (expandable), points, dates, courts
  - Plates table: plate, state, ticket count, violations (expandable)

**What DMV staff sees for each driver:**
- Driver license number
- Full name
- All violations within window (date, code, plate, agency, issuer)
- Points per violation and total
- ISA status badge (red = required, yellow = monitoring)

**Warning Band Tables:**
- Separate section showing near-threshold drivers and plates
- "Points to threshold" column makes it clear how close they are
- Supports proactive outreach workflow

### Step 4: Notice Sending â†’ Actual Enforcement

**Single Driver Notice (from License Violations page):**

2. Sees full violation history, points, ISA status

**What Gets Recorded:**

**Automatic vs Manual:**
- Currently: analyst-triggered (one-click from dashboard)

- Human review step preserved (preview before send)

## NOTICES & EMAIL FLOW: WHO SENDS WHAT
================================================================================

### Email Template (templates.html)
Professional HTML email matching NY DMV branding:
- Official header: "New York State Department of Motor Vehicles"
- Warning box: "Mandatory Action Required â€” install ISA within 30 days"
- Violation summary table with: license number, plate, code, description, total points
- Explanation of ISA requirement and thresholds
- Next steps: schedule installation, submit proof, maintain device
- Failure to comply warning
- Link to authorized installation locations

### API Endpoint: POST /api/dmv/isa/send-summary
```json
Request:
{
  "recipients": ["dmv@ny.gov", "vendor@isa.ny.gov"],
  "drivers": [{ "license_number": "...", "total_points": 24, ... }],
  "plates_count": 15,
  "send_real_email": false  // true in production
}

Response:
{
  "status": "ok",
  "message": "ISA notifications queued (demo mode)",
  "sent_to": ["dmv@ny.gov", "vendor@isa.ny.gov"],
  "summary": {
    "drivers_11_plus": 10,
    "plates_16_plus": 15,
    "emails_sent": 10
  }
}
```

### Who Receives What:
1. **DMV Enforcement Team**: Dashboard access, can send notices, receives summary reports
2. **Violator**: ISA notice email with installation instructions (via mailing address in production)
3. **Vendor**: CSV export of current ISA list for scheduling installations
4. **Local Courts**: Upload interface, see their violations reflected in system
